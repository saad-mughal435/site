ShopFloor API · OEE

OEE playground.

OEE = Availability x Performance x Quality - the headline manufacturing KPI for how much good output a line actually produced versus its theoretical best. This runs the same math as my open-source ShopFloor API, a Spring Boot service built next to a production MES. Every value is computed in your browser - change an input and the result updates instantly. Each factor is clamped to the 0-100% range and every division is guarded, so there is no Infinity or NaN, ever.

How it was computed

    The math (same as the API)

    Run time is planned time minus downtime, floored at zero. Availability is run time over planned time. Performance compares the ideal time the produced units should have taken at rated speed against the actual run time. Quality is good units over total units. Each factor is clamped to [0, 1]; multiply the three for OEE. Divide-by-zero anywhere (no planned time, no run time, no units, no rated speed) returns 0 for that factor instead of blowing up - this mirrors a real guard in the Java source.

    Java · OeeCalculator

    runTime      = max(0, planned - downtime)
availability = clamp01(runTime / planned)
idealMinutes = rated <= 0 ? 0 : total * 60 / rated
performance  = clamp01(idealMinutes / runTime)
quality      = clamp01(good / total)
oee          = availability * performance * quality
